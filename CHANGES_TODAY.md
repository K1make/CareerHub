# Изменения за 08.06.2026

Сегодня была проделана большая работа по доработке профиля кандидатов и работодателей. Вот полный список изменений:

### 1. Онбординг новых пользователей
- Добавлено всплывающее модальное окно (`OnboardingModal.jsx`) при первом входе для студентов и соискателей.
- Сделано обязательным заполнение номера телефона.
- Добавлено необязательное поле для ссылки на GitHub.
- Окно нельзя закрыть, пока не будет введён телефон. Данные автоматически сохраняются в базу.

### 2. Подтверждение образования (Дипломы)
- Добавлено поле `diploma_file` в БД (модель `User`).
- Настроен бэкенд (Django) для локального сохранения файлов.
- Frontend (React) переделан на использование `FormData` при сохранении профиля для корректной передачи файлов на сервер.
- В профиле добавлена возможность "Прикрепить диплом". Если диплом не прикреплен, система предупреждает пользователя, но не удаляет написанный им текст об образовании.
- В готовом профиле (как для самого пользователя, так и в карточке `CandidateProfilePage` для работодателей) появилась кликабельная синяя ссылка **«Смотреть диплом»**, по которой открывается загруженный документ.

### 3. Динамические списки в профиле
- В базу добавлены поля для хранения массивов JSON: `experience` (опыт работы), `projects` (проекты), `test_results` (тесты).
- **Опыт работы и Проекты**: 
  - Теперь они изначально пустые.
  - Появилась кнопка редактирования, которая позволяет добавлять неограниченное число мест работы и проектов.
  - В проектах можно указывать список технологий (стеки), которые отображаются красивыми бейджиками.
- **Результаты тестов**:
  - Блок изначально пустой.
  - Убрана возможность ручного редактирования. Данные будут автоматически заполняться после того, как человек сдаст тест.
- В контактах профиля теперь динамически подтягиваются актуальные Email, телефон и GitHub-ссылка.

### 4. Динамический бейдж уровня
- Уровень (Junior / Middle / Senior) больше не прописывается жестко в коде. 
- Теперь он высчитывается автоматически и отображается только если у пользователя есть сданные тесты. Берется лучший результат по тестам.
- Если тестов нет, бейдж скрыт. 

### 5. Синхронизация данных
- Переработан `AuthContext.jsx`. Теперь данные после изменения в профиле моментально синхронизируются по всему приложению без необходимости перезагружать страницу или выходить из аккаунта.
